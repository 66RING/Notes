# transformer概述

attention机制直观且简单的理解: 一个token和其他token的加权求和

$sum(Q K_i V_i)$

Q K V的关系

- QK: 对于单个Q, 给它赋上不同位置K的"权重"
- QK V: 对所有V加权求和


## 原理和机制

> [神级教程](https://www.bilibili.com/video/BV1q3411U7Hi/?vd_source=fa5227c06f0a0c9f870b406f10be1d31)

### attention

对于每个token(单词, x), 我们使用一个向量来表示, 如(x11, x12, x13, ...)。这个向量会用来索引一个词表, 从词表中获取单词的真正含义。多个单词拼在一起就是一个矩阵, 方便使用矩阵运算加速。如:

```
单词1, 单词2, 单词3, 单词4
x11,   x21,   x31,   x41
x12,   x22,   x32,   x42
x13,   x23,   x33,   x43
x14,   x24,   x34,   x44

----

单词1: x11, x12, x13, x14
单词2: x21, x22, x23, x24
单词3: x31, x32, x33, x34
单词4: x41, x42, x43, x44
```

我们需要训练对每个单词的attention, 因此引入一个"权重向量", 多个单词的权重矩阵在一起就成来权重向量, 即模型参数。对于一个单词, 矩阵相乘(注意顺序)就给每个元素附上了可训练的参数$w_{ij}$:

```
w11 w12 w13 w14
w21 w22 w23 w24

                  x11, x21
                  x12, x22
                  x13, x23
                  x14, x24

w11x11 + w12x12 + w13x13 + w14x14, w11x21 + w12x22 + w13x23 + w14x24
w21x11 + w22x12 + w23x13 + w24x14, w21x21 + w22x22 + w23x23 + w24x24

aka

Q1,   Q2
w1x1, w1x2
w2x1, w2x2

---------

x11, x12, x13, x14
x21, x22, x23, x24
                   w11, w12
                   w21, w22
                   w31, w32
                   w41, w42

Q1: x11w11 + x12w21 + x13w31 + x14w41, x11w12 + x12w22 + x13w32 + x14w42
Q2: x21w11 + x22w21 + x23w31 + x24w41, x21w12 + x22w22 + x23w32 + x24w42
```

需要QKV三个这样的矩阵初始都是通过$W \cdot X$得到, 其含义就是**每行表示一个权重可训练的单词, aka Q,K,V**

然后可以来看attention的公式: $attention(Q, K, V) = softmax(\frac{QK^T}{\sqrt{d}})V$, 我们来解释每个步骤的含义。首先发生的是QK相乘(注意顺序), 因为QKV都是来自同一个输入序列的, 所以每天元素背后指代的单词都相同，而$QK^T$的表示的就是: **一个单词和其他所有单词的联系**。比如, "它是一个漂亮的狗", 漂亮和狗就构成一种联系, transformer就是通过大量人类的文本来学习人类语言的模式。

> 注意这里QK是**内积**(元素按对应位置相乘再相加), 相当于一个向量转置后和另一个向量做矩阵乘法。

```
Q1
Q2
    x
      K1, K2


Q1K1, Q1K2
Q2k1, Q2K2
```


## 参数量计算

- 模型参数量: $N = 12h^2$
- 训练显存: $20B * N$
- KV cache: $4lb(s+n)h$
- 推理显存: $2B * N + KVcache$


### 模型参数量

> $12lh^2$粗略计算

- self attention: Q, K, V, O
    * 3个QKV的权重矩阵W和1个O的权重矩阵, 大小都是`[h, h]`, h表示hidden size。每个个有一个偏置矩阵`[h]`
    * $4 h^2 + 4h$
- MLP: 两个线性层
    * 两个线性层负责映射到其他范围, 每层各有一个偏执
    * `[h, 4h]`扇出再加上其偏执矩阵`[4h]`
    * `[4h, h]`扇入再加上其偏执矩阵`[h]`
    * $8h^2 + 5h$
- layernorm: attention和MLP各一个
    * 两个参数: 缩放和平移, 大小都是`[h]`
    * $2 \times 2h$
- 总的: $12h^2 + 13h$

> llama2-70B中h=8192, 则根据12lh^2计算大概有86层


### 训练过程显存占用分析

> 模型参数, 前向的激活值, 后向的梯度, 优化器状态

AdamW优化器为例, 用两个参数表示优化器状态。一个参数会产生一个梯度, 用到优化器里进行梯度更新，而优化器为了保证精度往往使用float32进行计算(混合精度)。所以显存占用组成如下:

1. 权重和梯度: 2B + 2B
2. 优化器中的梯度: 4B(把float16 cast成float32)
3. 优化器中的参数: 4B(cast成float32的权重) + 2x4B(优化器中的两个参数)
4. 总的: 20B

> llama2-70B意味着有70B个参数, 直接70 * 20 = 1400GB

#### 激活值分析

> 前向过程产生激活值, 要保存到反向传播时用来更新梯度。

梯度计算需要用到: 输入, 均值, 方差。对于输入有bsh(batch size, sequence len, hidden size)个元素, 对于均值和方差有bs个元素。

### 推理过程显存占用分析

在不考虑kv cache的情况下, 推理过程没有优化器, 没有梯度, 所以只有权重的2B数据。

#### KV cache

> 使用KV cache的推理分为两个阶段: 预填充(第一次处理一大段输入的prompt)和解码(一次生成一个token)。
>
> kv cache就是将$x \cdot W_k$的结果保存起来避免大量计算。

简单来说就是要保存所有输入的KV计算结果, 假设输入序列长s, 输出序列长n, 则总序列长s+n。总数据量就是lb(s+n)h。

需要存储KV的float16的数据则KV cache总显存占用为: $2 * 2 * lb(s+n)h = 4lb(s+n)h$
